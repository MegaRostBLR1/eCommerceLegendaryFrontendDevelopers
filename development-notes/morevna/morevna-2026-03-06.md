# Дата: 2026-03-06

- **Проблемы, Решения, Что было сделано:** Админ теперь может зайти в профиль любого пользователя по прямой ссылке, увидеть его текущие данные, изменить любые поля (включая роль) и сохранить изменения в базе данных.

Выпадашка с ролью выглядела ужасно, поправила её с помощью appearance: none. 
Реализовала подсветку ошибок - если одно из обязательных полей не заполнено. С помощью массива ошибок и классного селектора :has(.input-error).
Расширила тип UpdateUserDto теперь можно менять и роль юзера.
Сейчас у меня такая логика:
1 пользователь меняет одно поле
2 setUser обновляет только его
3 остальные поля копируются из прошлого state
4 при отправке идёт весь объект (реакт создаёт новый объект каждый раз)
И сама бы я пыталась отправить только часть данных (patch-подход), но это оказалось сложнее и полезла в документацию реакт https://react.dev/reference/react-dom/components/input. Где и нашла решение. Вообще вся работа современного разработчика состоит из того чтобы подобрать идеальные (уже существующие) решения для его проблемы, придумывать что-то своё, кажется, надо не так уж и часто.
Правда у этого подхода есть проблема - слишком много ререндеров формы (буквально при вводе каждой буквы!). Но реальной проблемой это будет при гораздо большей форме - моя маленькая и всё ок.
Вообще очень здорово и компактно на мой взгляд получилось: админ попадает в реальный кабинет юзера по id. Вся основная логика была реализована внутри ProfilePage - это переиспользование компонента.
Динамический роутинг: useParams очень помог подставлять в url id юзера.
Ввела проверку: admin или user - чтобы понять можно ли менять роль или нет.
Конечно сейчас очень неудобно - страницы со списком всех юзеров пока нет. И чтобы увидеть и проверить свою работу приходилось самостоятельно формировать url путь. Следующим этапом надо сделать эту страницу.
