## 3. Личные Feature Components (для презентации)
Ниже представлены два ключевых компонента, которые я готова детально разобрать на технической защите.

* ### Admin User Control & Role Management
Этот компонент является центральным для административной части приложения.
* **Реализация:** Я создала страницу управления пользователями с поиском по email и пагинацией. 
* **Техническая ценность:** Мною реализована возможность мгновенного редактирования данных пользователя (имя, отчество, фамилия, роль) через модальные окна. Я обеспечила корректную обработку асинхронных запросов к серверу и обновление локального состояния (state management), чтобы админ видел результат изменений без перезагрузки страницы.

* ### Order Lifecycle 
Я отвечала за весь путь пользователя от кнопки "Заказать" до просмотра истории своих действий.
* **Реализация:** Компонент включает в себя универсальную модалку заказа с валидацией (проверка даты и времени) и страницу "Orders", где заказы группируются и отображаются с учетом текущего статуса.
* **Техническая ценность:** Мною реализована архитектура управления состоянием заказов, которая обеспечивает синхронизацию данных между действием пользователя и историей в профиле. Я разработала систему динамической фильтрации (включая обработку категории "all"), где логика отображения адаптируется под роль пользователя (простой юзер или админ). Особое внимание было уделено работе с роутингом: использование динамических путей (например, /orders/:userId) позволило создать масштабируемое решение, при котором администратор может бесшовно переходить к истории заказов конкретного клиента, сохраняя контекст данных.
